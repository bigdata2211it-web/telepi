# AI Setup Guide: TelePi

Telegram bot for Pi Coding Agent. Node.js v20+, grammy framework.

## Quick start

```bash
git clone https://github.com/bigdata2211it-web/telepi.git
cd telepi
npm install
# create config at ~/.config/telepi/config.env:
#   TELEGRAM_BOT_TOKEN=<from BotFather>
#   TELEGRAM_ALLOWED_USER_IDS=<your telegram id>
npm start
```

## Files

| File | Purpose |
|------|---------|
| `dist/cli.js` | entry point |
| `dist/bot.js` | Telegram bot (grammy) |
| `dist/create-mirror.js` | bot mirroring |
| `dist/install/service-manager.js` | cross-platform service (systemd/nssm) |
| `telepi.sh` | Linux launcher |
| `telepi.ps1` | Windows launcher (PowerShell) |
| `install-windows.ps1` | Windows setup (downloads NSSM, creates service) |

## Telegram commands

`/mirror` — clone bot with another token. `/model <name>` — search models. `/new` — new session. `/sessions` — list/switch. `/abort` — stop streaming. `/retry` — retry last prompt.

## Architecture

User → Telegram → bot.js → pi-session.js → Pi Coding Agent API

Messages stream back with editMessageText (debounced). Queue system in chat-task-runner.js — inline ✖️🛑 buttons for cancel.

## Cross-platform

- Linux: systemd service (`systemctl --user enable telepi`)
- Windows: NSSM service (`telepi.ps1 install`)
- Mirror: service-manager.js selects systemd or nssm based on `process.platform`
